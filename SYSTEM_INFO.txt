================================================================================
  WHAT IS PROJECT SENTINAL?
================================================================================

Project Sentinal is an AI-powered crime intelligence platform built for the
Karnataka State Police. It centralises data from the crime records database,
FIR documents, and intelligence reports into a single real-time command
interface — enabling officers to investigate faster, predict hotspots, and
generate reports automatically.

Think of it as: A police "war room dashboard" that talks, listens, and thinks.

================================================================================
  TECH STACK
================================================================================

FRONTEND
  - React 18 + Vite
  - Vanilla CSS with dark-mode glassmorphism design
  - Hosted on Zoho Catalyst Static Hosting / AppSail
  - Real-time data via REST API calls to backend

BACKEND
  - Python 3.11 + FastAPI (async, production-grade)
  - Hosted on Zoho Catalyst AppSail (Docker container, port 9000)
  - SQLite database (sentinal.db) with Karnataka crime data

AI / ML STACK (ALL CATALYST-NATIVE)
  - Zoho Catalyst QuickML GLM-4.7-Flash   → Intelligence Q&A, reports
  - Zoho Catalyst QuickML VL-Qwen3.6-35B  → Vision/OCR analysis of FIR images
  - Zoho Catalyst Zia NLP                 → Text translation, TTS, STT
  - Zoho Catalyst Serverless Functions    → FIR OCR processing pipeline

  ⚠️ Groq has been fully removed. Zero external AI API calls.
     Everything runs on Zoho Catalyst native services.

DATA / RAG MEMORY
  - 94,717 FIR records embedded into vector memory (semantic search)
  - Source: master_dataset.jsonl (first half = 93,755 FIR records)
  - Source: investigation_narratives.json (1,000 curated narratives)
  - Embedding model: all-MiniLM-L6-v2 (built locally, served as .npy)
  - Format: backend/data/embeddings.npy (139 MB) + chunk_metadata.json

ML PREDICTION MODELS
  - hotspot_v2.joblib  → Crime hotspot prediction (RandomForest, 9 features)
  - risk_model.joblib  → Accused risk scoring (RandomForest, 5 features)
  - Both trained on Karnataka crime data from sentinal.db

SERVERLESS
  - functions/fir_ocr_processor/ → Node.js function that processes uploaded
    FIR PDFs using Catalyst Zia OCR + QuickML for structured extraction

================================================================================
  KEY FEATURES
================================================================================

1. DASHBOARD
   - Real-time crime statistics overview
   - Crime trend graphs by district, month, crime type
   - Active alerts and pending investigation cards

2. INTELLIGENCE Q&A (RAG-POWERED)
   - Ask natural language questions about any crime, case, or FIR
   - System retrieves the most relevant FIR records from 94k document memory
   - GLM-4.7-Flash generates a structured intelligence briefing
   - Conversation history maintained per session
   - Voice input/output via Zia NLP (STT + TTS)

3. CRIME HEATMAP
   - Geographic hotspot visualization across Karnataka districts
   - ML-predicted hotspots overlaid on actual crime data

4. CASE MANAGEMENT
   - Browse, search, filter all registered cases
   - Full case details: accused, victims, evidence, timeline
   - Add cases to an "Ops Board" for collaborative investigation

5. NETWORK / LINK ANALYSIS
   - Accused network graph: who knows whom, co-accused connections
   - Identify repeat offenders and gang networks

6. FINANCIAL CRIME TRACKING
   - Track stolen property values, UPI fraud transactions
   - CDR (Call Detail Record) analysis

7. ALERTS & DARKWEB INTELLIGENCE
   - Real-time alert feed from integrated data sources
   - Darkweb intelligence monitoring module

8. FIR OCR PROCESSOR
   - Upload a FIR PDF → Zia OCR extracts text → QuickML structures data
   - Extracted FIR fields: crime type, accused, location, date, sections

9. AI REPORTS (PDF GENERATION)
   - Generate formatted PDF intelligence reports for any case
   - Powered by GLM-4.7-Flash + ReportLab

10. HOTSPOT PREDICTION
    - Input: Police station ID, month → Output: Predicted hotspot risk
    - Trained RandomForest model, F1/AUC: ~1.0 on training data

================================================================================
  LOCAL DEVELOPMENT SETUP
================================================================================

  Prerequisites: Python 3.11+, Node.js 18+, pip

  1. Clone repo:
     git clone https://github.com/brovk2008/Sentinal.git
     cd Sentinal

  2. Install backend deps:
     cd backend
     pip install fastapi uvicorn httpx numpy scikit-learn joblib pandas reportlab aiofiles beautifulsoup4 python-dotenv python-multipart
     cd ..

  3. Build RAG embeddings (one-time, ~30 min, needs sentence-transformers):
     pip install sentence-transformers
     python scripts/add_fir_data.py
     python scripts/build_embeddings.py

  4. Train prediction models:
     python scripts/train_hotspot_v2.py
     python scripts/train_models.py

  5. Start backend:
     cd backend
     python -m uvicorn main:app --host 0.0.0.0 --port 9000 --reload

  6. Start frontend (new terminal):
     cd frontend
     npm install
     npm run dev

  7. Open http://localhost:5173
     Login: demo@sentinal.ksp / Sentinal@2024

================================================================================
  IMPORTANT NOTES FOR JUDGES
================================================================================

  1. The QUICKML_KEY must be set as an environment variable in the Catalyst
     AppSail console for AI features to work. Without it, the system returns
     "Catalyst QuickML is not configured" for AI queries.

  2. The database (sentinal.db) contains synthetic Karnataka crime data used
     to demonstrate the dashboard, heatmap, and ML prediction features.

  3. The RAG system (94k FIR records) is pre-built and loaded at startup
     from backend/data/embeddings.npy. No internet required for retrieval.

  4. Voice features (STT/TTS) require the Catalyst Zia NLP endpoints to be
     accessible. These are production Catalyst APIs.

  5. The system is fully bilingual-aware: FIR records in Kannada are parsed
     for structural fields (crime type, location, date) even though the
     narrative text may not be in English.
